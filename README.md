# LP Meeting Copilot

기관 LP가 운용사 미팅을 `사전 준비 -> 미팅 노트 -> 최종 보고서` 흐름으로 처리하는 GitHub Pages용 정적 웹앱입니다.

## 사용 방법

1. GitHub Pages URL에 접속합니다.
2. 왼쪽 아래 `설정`에서 이름, 부서, Gemini API Key, Gemini 모델을 입력합니다.
3. `사전 준비`에서 IM 파일을 업로드하거나, IM이 없으면 운용사명/펀드명 또는 대출명/지역/자산군/전략/섹터/구조/딜 메모를 입력합니다.
4. `브리프 생성`을 눌러 Fund Snapshot, 시장 맥락, 예상 Q&A, 리스크, Red Flag를 생성합니다.
5. `미팅 노트`에서 질문별 운용사 답변, 내부 메모, 추가 검토 필요 여부를 기록합니다.
6. `최종 보고서`에서 보고서를 생성하고 Markdown 또는 DOCX로 내려받습니다.

## API Key 보안

- Gemini API Key는 코드나 저장소에 포함되지 않습니다.
- API Key는 localStorage/sessionStorage에 저장하지 않습니다.
- API Key는 현재 브라우저 탭 메모리에만 적용됩니다.
- 새로고침하거나 탭을 닫으면 API Key를 다시 입력해야 합니다.

## 로컬 저장 정책

- localStorage에는 미팅 데이터와 비민감 프로필만 저장합니다.
- 미팅 저장 대상: 현재 미팅, 최근 미팅 목록, 질문별 답변, 자유 메모, 생성 결과.
- 프로필 저장 대상: 이름, 부서, 선택 모델, 미팅 데이터 저장 켜짐/꺼짐 설정.
- 저장하지 않는 항목: Gemini API Key.
- 설정창에서 `미팅 데이터 localStorage 저장`을 끌 수 있습니다.
- 설정창에서 `전체 삭제`를 누르면 브라우저에 저장된 현재 미팅과 최근 미팅 목록을 지웁니다. 프로필은 유지됩니다.
- 설정창에서 `백업 후 저장소 비우기`를 누르면 JSON 백업 파일을 내려받은 뒤 localStorage의 미팅 데이터를 삭제합니다. 프로필은 유지됩니다.

## IM 처리 방식

- 지원 파일: PDF, TXT, PNG, JPG.
- 텍스트 PDF는 브라우저에서 텍스트를 추출해 Gemini에 전달합니다.
- 이미지 PDF 또는 스캔본은 앞부분·중간·후반·마지막 페이지를 이미지로 변환해 Gemini Vision에 전달합니다.
- 큰 IM은 앞부분만 보지 않고 핵심 키워드 페이지, 중간·후반 페이지, 마지막 페이지를 함께 샘플링해 읽습니다.

## 시장/뉴스 검색

Gemini Google Search grounding을 사용합니다. 자산군별 기본 검색 체크리스트를 먼저 적용하고, IM과 입력값에서 발견된 특이사항을 추가 검색 단서로 사용합니다.

- PE: 거시/금리, Buyout 시장, Exit/M&A/IPO, 레버리지론, GP 뉴스, 파이프라인/포트폴리오 뉴스.
- PD: 크레딧 스프레드, 부실률, 리파이낸싱, 차주/담보/코버넌트, GP 크레딧 뉴스.
- 부동산: 금리, PF 조달환경, 임대/공실/거래/캡레이트, 권역 거래사례, 시공사/보증기관/차주 뉴스.
- 인프라: 정책/규제, 수요, 요금체계, EPC/O&M, 사업권, 유사 거래 사례.
- 상품금융: 항공기/선박 시황, 운임/리스료, 잔존가치, Balloon 상환, 차주/리스이용자 뉴스.

## GitHub Pages 배포

1. 이 폴더의 파일을 GitHub 저장소에 업로드합니다.
2. GitHub 저장소에서 `Settings -> Pages`로 이동합니다.
3. `Build and deployment`를 `Deploy from a branch`로 선택합니다.
4. Branch는 `main`, Folder는 `/root`를 선택합니다.
5. 배포가 완료되면 제공되는 GitHub Pages URL로 접속합니다.

서버 실행 명령이나 환경변수는 필요하지 않습니다.
